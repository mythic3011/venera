# M27 Source Runtime SDK and Normalization Layer

## Goal

- Turn the source JavaScript runtime from a bare script executor into a constrained source adapter SDK.
- Provide common runtime helpers for HTTP, HTML parsing, JSON validation, typed source errors, tag normalization, OpenCC conversion, diagnostics redaction, and source capability checks.
- Reduce duplicated parsing/error-handling code inside individual source scripts.
- Prevent source scripts from throwing raw JavaScript errors into app UI.
- Preserve existing source behavior through compatibility wrappers.

## Problem

Current source runtime behaves too much like a raw JavaScript engine.
It can execute source scripts, but common platform responsibilities are not clearly owned:

- HTML parsing
- API JSON response validation
- `code != 200` response handling
- missing field handling
- pagination shape validation
- chapter grouping
- tag/name normalization
- simplified/traditional Chinese conversion
- network retry / timeout policy
- auth/session redaction
- diagnostics safety
- typed error mapping
  As a result, every source script must hand-roll behavior. When API response shape changes or returns a non-success domain code, script logic can throw raw errors such as:

```text
ReferenceError: res is not defined
TypeError: cannot read property 'name' of undefined
NoSuchMethodError: Class 'String' has no instance getter 'entries'

This is not a source-platform model. It is only script execution.

Scope

* source runtime SDK helpers
* source script compatibility wrapper
* typed source result model
* typed source error model
* HTTP response validation helpers
* HTML parser helper
* JSON schema/path helper
* tag/name normalization helper
* optional OpenCC conversion helper
* diagnostics redaction helper
* source capability declaration
* plugin API versioning

Out of scope:

* no ReaderNext route semantics change
* no ReaderNext render pipeline rebuild
* no appdata migration change
* no cache migration change
* no cookie migration change
* no source repository management UI change
* no source package registry change
* no source install/update workflow rewrite
* no bypass logic for third-party source restrictions
* no network signing/auth redesign

Hard Rules

1. Source runtime must expose a versioned SDK surface.
2. Source scripts must not depend on unversioned app internals.
3. Source scripts must not hand-roll common response validation where SDK helpers exist.
4. code != 200 API responses must become typed source errors.
5. Missing required fields must not throw raw JavaScript TypeError.
6. Undefined variables inside source wrappers must become typed runtime failures with source/stage context.
7. HTML and JSON source modes must both be supported.
8. OpenCC conversion must be application-owned or SDK-owned, not duplicated across scripts.
9. Tag normalization must be deterministic.
10. Normalization must not destroy original source text.
11. Source runtime diagnostics must redact tokens, cookies, signatures, device IDs, pseudo IDs, and auth headers.
12. Source script contents must not be dumped into diagnostics by default.
13. Source runtime must return typed results to Dart/app layer.
14. Source runtime must expose visible typed UI error states, not silent empty UI.
15. Source runtime must support compatibility mode for existing scripts.
16. SDK helpers must be additive first; existing scripts must not be broken in M27-A.
17. Source capability declaration must be explicit.
18. Source scripts must not be allowed to mutate global app settings directly.
19. Runtime must separate source-specific logic from application-owned policy.
20. M27 must not change ReaderNext cutover/fallback semantics.

Runtime Responsibility Model

Application
- owns storage
- owns UI
- owns settings
- owns diagnostics policy
- owns language preference
- owns source install/update policy
- owns cache policy
Source Runtime SDK
- provides safe capabilities to scripts
- validates common response shapes
- maps source failures to typed errors
- normalizes text/tags where requested
- redacts diagnostics
- controls runtime API version
Source Script
- implements source-specific URL construction
- maps source-specific payload to common comic/chapter/page models
- declares capabilities
- avoids direct app internals

Source Runtime SDK Surface

1. HTTP Helper

runtime.http.request({
  method: "GET",
  url,
  headers,
  query,
  body,
  timeoutMs,
  responseType: "json" | "html" | "bytes" | "text",
  redact: ["authorization", "cookie", "x-auth-signature"]
})

Expected behavior:

* applies app network policy
* supports timeout
* returns typed response
* redacts sensitive headers in logs
* does not throw raw network exception unless wrapped

Typed response:

{
  ok: true,
  status: 200,
  headers: {},
  body: {},
  rawBodyPreview: "<redacted or limited>"
}

Typed failure:

{
  ok: false,
  error: {
    code: "NETWORK_REQUEST_FAILED",
    message: "Request failed",
    stage: "loadInfo"
  }
}

2. API JSON Helper

runtime.json.expectApiSuccess(response, {
  successCode: 200,
  codePath: "code",
  messagePath: "message",
  resultsPath: "results",
  stage: "loadInfo"
})

Expected behavior:

* if code == 200, returns results
* if code != 200, throws/returns typed SourceBlocked or SourceUnavailable
* preserves source message as user-safe message where allowed
* does not allow downstream code to read missing results.comic.name

Example mapping:

code=210 -> SOURCE_BLOCKED or SOURCE_REQUIRES_OFFICIAL_APP
code=401 -> SOURCE_AUTH_REQUIRED
code=403 -> SOURCE_FORBIDDEN
code=404 -> SOURCE_NOT_FOUND
network failure -> SOURCE_UNAVAILABLE
schema mismatch -> SOURCE_SCHEMA_INVALID

3. JSON Path Helper

runtime.json.required(obj, "comic.name", {
  stage: "loadInfo",
  code: "SOURCE_SCHEMA_INVALID"
})

Expected behavior:

* returns value if present
* if missing, returns typed schema error
* never throws raw cannot read property 'name' of undefined

Optional path:

runtime.json.optional(obj, "comic.last_chapter.name", null)

Array helper:

runtime.json.array(obj, "groups.default.list", {
  stage: "loadChapters"
})

4. HTML Parser Helper

const doc = runtime.html.parse(htmlText);
const title = runtime.html.text(doc, "h1.title", {
  required: true,
  stage: "loadInfo"
});

Expected behavior:

* supports CSS selector extraction
* typed missing selector error
* common whitespace normalization
* no raw parser errors leaking to UI

5. Text Normalization Helper

runtime.text.normalizeWhitespace(text)
runtime.text.stripHtml(text)
runtime.text.decodeEntities(text)

6. OpenCC / Language Conversion Helper

runtime.text.opencc(text, {
  from: "s2t",
  mode: "ui"
})

or source model level:

runtime.normalize.comicInfo({
  name,
  tags,
  authors,
  languagePolicy: "app_ui"
})

Policy:

* original source text must remain available
* converted display text may be used for UI/search
* conversion must be deterministic
* app language preference controls default display conversion
* source scripts should not each embed their own OpenCC logic

7. Tag Normalization Helper

runtime.tags.normalize([
  "爱情",
  "校园",
  "百合"
], {
  locale: "zh-Hant",
  preserveOriginal: true
})

Output:

[
  { original: "爱情", display: "愛情", normalizedKey: "愛情" },
  { original: "校园", display: "校園", normalizedKey: "校園" }
]

Rules:

* preserve original tag
* derive display tag
* derive normalized key
* support app UI locale
* avoid duplicate tags after conversion
* maintain stable ordering

8. Typed Error Helper

runtime.error.sourceBlocked({
  code: "SOURCE_REQUIRES_OFFICIAL_APP",
  message: "Source rejected this request",
  stage: "loadInfo"
})

Common error classes:

SOURCE_BLOCKED
SOURCE_AUTH_REQUIRED
SOURCE_FORBIDDEN
SOURCE_NOT_FOUND
SOURCE_UNAVAILABLE
SOURCE_SCHEMA_INVALID
SOURCE_RATE_LIMITED
SOURCE_SETTINGS_INVALID
SOURCE_CAPABILITY_UNSUPPORTED
SOURCE_RUNTIME_EXCEPTION

9. Diagnostics Helper

runtime.diagnostics.info("source.loadInfo.start", {
  sourceKey,
  comicId: runtime.redact.id(comicId)
});

Rules:

* diagnostics must be structured
* raw headers must be redacted
* raw URLs must be redacted where query contains secrets
* device IDs/signatures/tokens must never persist unredacted
* source script content is never logged by default

10. Source Capability Declaration

Each source script should declare:

module.exports.capabilities = {
  apiVersion: "source-runtime-v2",
  modes: ["api", "html"],
  supportsSearch: true,
  supportsTags: true,
  supportsLogin: false,
  supportsPagination: true,
  supportsChapterGroups: true,
  supportsDirectPageUrls: true,
  requiresAuth: false
};

Runtime must validate capability declaration.

Compatibility Strategy

M27 should be staged. Do not rewrite all sources at once.

Compatibility Mode

Existing scripts continue to run through a wrapper:

legacy script output -> adapter wrapper -> typed SourceResult / SourceError

Wrapper responsibilities:

* catch raw JS exceptions
* map missing fields to typed schema error
* redact diagnostics
* convert known dynamic settings shape errors to SOURCE_SETTINGS_INVALID
* keep behavior unchanged when output is valid

SDK Mode

New or migrated scripts use runtime.* helpers directly.

Source scripts can declare:

apiVersion = "source-runtime-v2"

Runtime behavior:

* v1 scripts: compatibility wrapper
* v2 scripts: SDK contract enforced

M27-B: Typed Source Result and Error Model

Add Dart-side typed result models:

sealed class SourceRuntimeResult<T> {}
class SourceRuntimeSuccess<T> extends SourceRuntimeResult<T> {
  SourceRuntimeSuccess(this.value);
  final T value;
}
class SourceRuntimeFailure<T> extends SourceRuntimeResult<T> {
  SourceRuntimeFailure(this.error);
  final SourceRuntimeError error;
}
class SourceRuntimeError {
  const SourceRuntimeError({
    required this.code,
    required this.stage,
    required this.userMessage,
    this.diagnosticMessage,
  });
  final String code;
  final String stage;
  final String userMessage;
  final String? diagnosticMessage;
}

Add mapping from JS exceptions:

TypeError -> SOURCE_RUNTIME_EXCEPTION or SOURCE_SCHEMA_INVALID
ReferenceError -> SOURCE_RUNTIME_EXCEPTION
dynamic settings shape mismatch -> SOURCE_SETTINGS_INVALID
code != 200 -> SOURCE_BLOCKED / SOURCE_UNAVAILABLE / SOURCE_AUTH_REQUIRED

M27-D: JS Compatibility Wrapper

Wrap existing source calls:

dynamicSettings
loadInfo
loadChapters
loadPages
search

Wrapper must:

* catch raw exceptions
* attach sourceKey and stage
* map to typed error
* prevent silent empty UI
* preserve valid outputs

Example:

async function safeCall(stage, fn) {
  try {
    const result = await fn();
    return { ok: true, result };
  } catch (e) {
    return {
      ok: false,
      error: runtime.error.fromException(e, { stage })
    };
  }
}
```
